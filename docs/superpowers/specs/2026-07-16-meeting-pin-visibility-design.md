# 일정 미정 모임의 지도 핀 노출 (#158) — 백엔드 설계

## 문제

현재 핀 조회는 `meetings.status = 'open'`인 모임에도 `meeting_schedules`에
`scheduled` 상태이면서 `visible_until >= now()`인 일정이 하나 이상 있어야만 지도와
핀 목록에 노출한다. 따라서 "일정 미정"으로 생성한 정상적인 모임은 장소와 이미지가
있어도 발견할 수 없다.

## 새 계약

- 모임 핀의 발견 가능 여부는 일정이 아니라 모임 수명으로 결정한다.
- `meetings.status = 'open'`이고 `meetings.deleted_at IS NULL`인 모임은 일정이 없거나,
  모든 일정이 만료·완료·취소됐어도 지도와 핀 목록에 노출한다.
- 모임은 방장이 명시적으로 모집 마감(`closed`)하거나 삭제/취소할 때까지 살아 있다.
- 일정 미정 모임은 생성 직후 지도에 보이고, `open`이면 신규 사용자도 참여할 수 있다.
  참여자는 기존 일정 추가 API로 이후 일정을 만든다.
- 강퇴 사용자와 서로 차단된 사용자는 기존과 같이 해당 모임 핀을 볼 수 없다.

## 구현 결정

`PinRepository.findMapPins`와 `findListPins`의 meetings JOIN에서 활성 일정 `EXISTS`
서브쿼리를 제거한다. 두 조회는 계속 `open`, 미삭제, 뷰어 강퇴 제외, 차단 제외 조건을
공유한다. `MeetingService.join`도 `open` 상태만 참여 조건으로 사용해, 발견한 일정 미정
모임에 참여한 사용자가 기존 일정 추가 API를 쓸 수 있게 한다. 일정의 캘린더·목록·생성과
정기 일정 생성 로직은 변경하지 않는다.

## 범위 밖

- DB 스키마·마이그레이션, REST 응답 형식, 프런트 지도 마커 렌더링은 변경하지 않는다.
- 지도 viewport 밖의 핀을 자동으로 재중심화하는 UX는 별도 문제다.
- 참가자용 일정 생성 화면은 별도 프런트 기능이며, 이 변경은 기존 API 권한만 연결한다.

## 회귀 검증

`PinRepositoryIntegrationTest`에서 일정 없음과 만료 일정만 가진 open 모임이 지도 및
목록 조회 모두에 포함됨을 고정한다. `closed`, soft-deleted, 강퇴, 차단 모임은 계속 두
조회에서 제외됨을 함께 검증한다. `MeetingServiceTest`에서는 일정이 없는 open 모임도
신규 참여자를 추가하고 그룹 채팅방에 연결함을 검증한다.
